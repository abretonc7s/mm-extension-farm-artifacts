# Review Learnings — PR #42009

- **`ext_navigate_hash` strips a leading slash but not a leading `#`.** Passing `#/perps/market/BTC` produced the URL `home.html#/#/perps/market/BTC` (concatenation, no PerpsLayout mount). Passing the path without the `#/` prefix (`perps/market/BTC`) is correct. Source: `temp/recipes/lib/ext-bridge.js:925-931`.
- **`PerpsHome` route target = `#/?tab=perps`, not a `/perps/*` route.** Calling the existing `perps/navigate-perps-tab` flow does NOT mount `PerpsLayout` — the route is the wallet home with a tab query, not under `/perps/*`. To exercise persistence/redirect, use `ext_navigate_hash` with `perps/market/<symbol>` directly. Easy to miss because the flow name suggests a `/perps` mount.
- **Recipe assert engine has no `is_null` operator.** Use `falsy` (or `eq` with `value: null`) instead. `not_null`, `truthy`, `falsy`, `eq`, `neq`, `gt/lt/gte/lte`, `length_*`, `contains`, `matches`, `one_of`, `exists` — that's the full list (`temp/recipes/lib/assert.js`).
- **`extension-core/unlock-wallet` flow asserts `account-menu-icon` visible on exit** — when the slot is parked on a non-home screen (e.g. left over `/perps/market/ETH` from a previous probe), `verify-unlocked` fails with `Expected visible=true, got false`. Either run `ext_navigate_hash` to `''` first, or guard against entering with the wrong screen.
- **Module-level state in `ui/helpers/perps/in-app-leave-marker.ts`.** Bridges the `useLayoutEffect` cleanup (writes timestamp) → next mount's `componentDidMount` (reads). The 1500ms decay is a magic number duplicated implicitly between writer and reader. The marker also has a `__resetPerpsInAppLeaveMarkerForTests` escape hatch only for the unit suite — the live recipe relies on time-decay.
- **AC5 (TTL > 5min) cannot be exercised in live CDP** because `setLastVisitedPerpsRoute` always stamps `Date.now()`. A debug-only `setLastVisitedPerpsRouteWithTimestamp` would make this testable; absent that, unit tests are the only lever.
- **AC2/AC6 (popup teardown + popup-vs-fullscreen parity)** are out of reach when the CDP session attaches to `home.html` only — opening the popup target requires a user gesture. Mark UNTESTABLE in the coverage matrix and route to the `useLayoutEffect` cleanup ordering review + the unit suite.
- **`recipe-issues-review.md` flags 1 unrelated background error per run** (`No metadata found for 'autoLockTimeLimit'`). It is preferences-controller noise, not PR-related — the auto-issues review correctly classes it as "review" (informational), not "gating".
- **`unset GH_TOKEN` is required for any `gh` write/POST** on this slot — the env var token lacks `repo:write`. Forgot once; the listing/diff calls are read-only and worked, but any review/comment post would fail without the unset.
- **`yarn lint:tsc` exits 0 silently when clean** — empty stdout/stderr with exit 0 is the success signal. Don't read it as "command didn't run".
