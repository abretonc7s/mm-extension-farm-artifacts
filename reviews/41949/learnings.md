# Learnings — PR #41949 review

- **`perps/open-order-form` flow lives in `temp/agentic/recipes/`, not `temp/recipes/`.** Author's task-local recipe assumes the canonical shared catalog; a fresh review slot has to copy `temp/agentic/recipes/domains/perps/flows/open-order-form.json` into `temp/recipes/domains/perps/flows/` before the recipe will run. Consider promoting this flow upstream so reviewers don't trip on it.
- **i18n key `perpsOpenLong` resolves to "Open long $1" (lowercase 'l'), not "Open Long $1".** Author's `ac5-valid-amount` subflow defaulted `sideLabel: "Long"` which mismatched. `eq` operator is case-sensitive — switched default to lowercase `"long"`. Worth pushing back upstream.
- **Recipe halt-on-failure cascades through subflow chains.** When `ac5-valid-amount` failed (case mismatch), all subsequent subflows (`ac2-select-on-focus`, `leverage-keyboard`, `ac1-limit-autofocus`, `ac7-enter-submit-success`) were skipped because they sit on the `next:` chain. The trace records 4/5 visible, which masks the fact that 4 ACs went un-exercised. Recommend authoring with parallel `entry` points or independent `call` orchestration so one failure doesn't shadow downstream coverage.
- **Tab key sequence is not exercisable without explicit Tab keydown dispatches.** Recipe assertion patterns rely on programmatic focus + select state; native Tab order is silently assumed correct. AC4 ended UNTESTABLE despite the slot being live.
- **CDP websocket handshakes refuse loopback origins by default on Chromium 140.** Direct websocket calls fail with `403 Forbidden — Use --remote-allow-origins=*`. Always go through `validate-recipe.js` (which uses CDP HTTP target endpoints) or `temp/recipes/status.ts` rather than raw `websocket.create_connection`.
- **`auto-issues` review status `review` is informational only.** A 9/9 PASS recipe run still produced 38× home "Unknown action Object" + 7× SW HttpRequestError AbortError + 7× SW Sentry-not-init. These are pre-existing background noise on this slot and do not gate the verdict — but the runner's `recipe-issues-review.md` summary is the authoritative place to confirm none promoted to `gating`.
- **mobile-extension-map mobile path:** `app/components/UI/Perps/hooks/useMinimumOrderAmount.ts` is the canonical reference for min-order-size resolution. Extension's `PERPS_MIN_MARKET_ORDER_USD = 10` constant is a deliberate stopgap with explicit TODO; review of any "min order size" PR should consult both files.
- **AC7 positive subflow uses state-based wait on `perpsGetPositions`, not DOM polling.** Resolves in ~3-5s vs prior 45s DOM-only timeout. State-wait pattern (`eval_async` polling controller state) is the right primitive for "await live submission outcome" — copy this pattern for similar e2e proofs.
- **`metaRPCClientFactory` is a single point of failure for *all* background-worker promise settling.** Any change there (like the empty-message guard in this PR) is high-blast-radius even when the diff is small. Always require a regression test on this file.
- **`fix: address review comments on PR #...` commit message convention.** Author squashes review-feedback responses into a single commit titled by PR number. Useful for quick `git log --grep` filtering when checking whether a `CHANGES_REQUESTED` review was addressed without diffing every commit.
