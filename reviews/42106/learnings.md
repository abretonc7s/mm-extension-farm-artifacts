# Learnings — PR #42106 review

- **Synthetic `MouseEvent` does NOT trigger lightweight-charts crosshair.** Dispatching `mouseenter`/`mouseover`/`mousemove` on the chart's `[data-testid="perps-candlestick-chart"]` element does not surface a crosshair tooltip. The library either binds canvas pointer-events directly with hit-testing or treats untrusted events as inert. Future recipes that need to validate canvas-rendered overlays should call into the lightweight-charts API via a `window`-exposed handle (`chart.subscribeCrosshairMove` + `chart.timeScale().getVisibleRange()`) rather than dispatching DOM mouse events.
- **Recipe `assert` composite uses `all`/`any`/`none`, not `and`/`or`.** Spent one failed live run discovering `Unknown operator: undefined` was thrown by `lib/assert.js:137`. The schema is in `lib/assert.js:73-83`. Also: there is no `is_true`/`is_false` operator — use `eq` with value `true`/`false`, or `truthy`/`falsy`.
- **`validate-recipe.js` needs `--artifacts-dir <abs-path>` for one-off recipes outside `domains/`.** Without it, `screenshot` actions try to mkdir `/artifacts` (root) and fail with ENOENT. The task-local recipe at `temp/tasks/review/<id>/artifacts/recipe.json` must be invoked with an explicit `--artifacts-dir`.
- **`hour12: false` in V8 `Intl.DateTimeFormat` resolves to `hourCycle: 'h23'` for en-US.** Verified at runtime in this slot: midnight renders `"00:00"`, not `"24:00"`. Cursor-bot's repeated "midnight displays as 24:00" claim is a stale ICU-spec misreading. Worth pre-empting future reports of this in any PR that adds 24h `Intl` formatters.
- **Czech (`cs`) ICU month-downgrade is real.** When you ask for `month: 'short'` combined with other fields in one `Intl.DateTimeFormat`, ICU may emit a numeric month part instead of the abbreviation. The PR's `formatWithMonthPatch` workaround using `formatToParts` + standalone `monthShort` formatter is the right shape and worth remembering for any locale-aware date code in this codebase.
- **`getIntlLocale` selector lives at `ui/ducks/locale/locale`** and is the canonical source of UI locale for components that need to thread locale into rendering libs. Prefer it over `navigator.language` once a chart/component is inside the React tree (lightweight-charts' `tickMarkFormatter` callback passes `chartLocale` as the third arg, sourced from `localization.locale`).
- **Module-level `Intl.DateTimeFormat().resolvedOptions().timeZone` capture is a common mobile-aligned pattern**, but it freezes the user's TZ at script load. Acceptable for chart contexts where TZ change requires a session reload; flag in review when any per-render TZ-sensitive code might want a fresh read.
- **`recipe-issues-review.md` auto-flags console errors that may be unrelated to the recipe.** Saw a pre-existing React `setState on unmounted component` warning in `PerpsView` during navigation that was unrelated to the formatter PR. Always read `recipe-issues.json` to decide whether to treat as gating or noise.
- **For perps fullscreen reviews, `perps/prime-perps-state` + `perps/navigate-to-market-detail` is the cheapest path to "chart visible" state.** Tutorial-modal handling is already built in. Pre-conditions `wallet.unlocked` + `perps.feature_enabled` cover the basic gate.
- **lightweight-charts v5 passes `tickMarkType` as a numeric enum (`0..4`)** to `tickMarkFormatter`, while older docs still reference the string variant. Code that supports both (this PR) is more robust against lib upgrades. The numeric mapping `['Year', 'Month', 'DayOfMonth', 'Time', 'TimeWithSeconds']` is documented inline in `chart-utils.ts:347-352`.
