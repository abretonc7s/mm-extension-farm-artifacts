# Learnings — PR #42101 review

- `ext_navigate_hash` builds `home.html#/<hash-after-stripping-leading-slash>`. Passing `#/perps/withdraw` produces `home.html#/#/perps/withdraw` (double hash). Pass hashes **without** the `#/` prefix: `perps/withdraw`. `hash` values are joined onto the already-present `#/`.
- For order-entry navigation, don't hand-roll the hash — use `navigate` with `target: "PerpsOrderEntry"` + `params: { symbol, direction }`. `resolveNavigateParams` in `lib/ext-bridge.js` builds the full `#/perps/trade/<symbol>?direction=<dir>&mode=new` for you and waits for DOM settle.
- The `assert` mini-language supports `all` / `any` / `none` (not `and` / `or`). Scalar ops include `truthy` / `falsy` — there is no `is_true` / `is_false`. Save regex-match results into a `save_as` object then assert with `field + truthy`.
- `{{vars.X.field}}` interpolation inside `eval_sync.expression` is a *string* substitution — you must quote with single-quotes in the JS (`'{{vars.X.text}}'`) or you'll inject raw tokens into JS and blow up at runtime.
- When the fixture hands you the exact scenario the PR targets, lean into it. The slot had `availableBalance:"0"` / `availableToTradeBalance:"29.67"` already — no setup call needed, the coverage claim then becomes unambiguous.
- Order-entry's `[data-testid="submit-order-button"]` is always present; the "disabled when balance zero" state is visually distinct but not reflected as `disabled` attribute reliably. For a stronger AC1 assertion, probe the text "Available to trade" node instead of the submit button's `.disabled` property.
- The repo's `yarn lint:tsc` exits silently on success — no stdout. A 0 exit code with empty output is a pass. Don't wait 10 minutes expecting "OK" text.
- Perps telemetry property `HAS_PERP_BALANCE` is now sourced from `getTradeableBalance`; if reviewing future changes in this area, any regression that reroutes telemetry back to `availableBalance` would silently under-report funded-but-only-spot-USDC accounts.
- Withdraw surfaces on extension (`perps-withdraw-page.tsx:97`, `perps-withdraw-balance.tsx:22-24`) are the canonical "withdrawable-only" readers. Any future PR that replaces these with `getTradeableBalance` would regress AC3; worth flagging in follow-up reviews.
- Mobile uses `adaptAccountStateFromSDK(perpsState, spotState)` to merge balances *before* they reach the UI; extension delegates that merge to controller v4 and keeps both fields separate in the UI. When reviewing mobile ↔ extension parity, don't look for a helper mirroring `getTradeableBalance` on mobile — it won't exist; the merge lives in the adapter.
- `@metamask/perps-controller@4.0.0` has an identical transitive dep list to `3.1.1` (same checksum line changes but no `dependencies:` edits in yarn.lock). That's why the PR didn't regenerate lavamoat — the module graph didn't change. Future bumps that add peer deps WILL require `yarn lavamoat:auto`.
